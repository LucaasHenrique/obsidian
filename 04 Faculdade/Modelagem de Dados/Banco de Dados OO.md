- Inspirados em orientação a objetos.
- Cada objeto possui um ID.

Nesse modelo o banco e viso como uma coleção de objetos, que são elementos de software reutilizaveis com recursos e metodos associaddos.

Os dados só podem ser manipulados pelo metodos definidos pela classe a que pertecem.

OID ➡ Cada objeto armazenado no banco recebe uma identidade unica e imutavel chamada OID. Este é invisivel para o usuario externo, mas é usado pelo sistema para gerenciar as referencias ao objeto.

Objetos complexos: Esses bancos suportam representaçoes diretas de objetos complexos, no qual, um atributo pode conter valores não atômicos, como conjuntos, listas ou vetores. Isso significa que eles nao seguem as formas normais usadas no bancos relacionais.
